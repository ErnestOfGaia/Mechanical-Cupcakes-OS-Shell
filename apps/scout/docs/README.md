# Scout Protocol Working Docs

Status: active

This folder contains the current working documents for Scout Protocol. The older `Scout Protocol Prototype Notes` folder is treated as source material and research context. When a decision becomes active, summarize it here in `docs/` instead of relying on raw notes.

## Working Areas

- `product/` - product requirements, user journeys, glossary, and prototype goals.
- `specs/` - implementation-facing specs for local prototype slices.
- `design/` - interface principles, wireframes, visual direction, and experience notes.
- `protocol/` - protocol message shapes, envelope rules, and capability taxonomy.
- `decisions/` - ADRs for architectural/product decisions.
- `process/` - preflight checklists, testing rituals, and repeated issue notes.

## Current Prototype Target

Garage v0.1 is a local prototype:

- One local Garage UI.
- One simulated peer node.
- A Walkie Talkie query-response loop.
- Saved agent candidates on a Whiteboard.
- Network Activity log showing message events.

Out of scope for v0.1: real blockchain settlement, real gossip, tokenomics, guild governance, production deployment, and autonomous scout agents.
