# Scout Local Issue Backlog

Status: active

This folder holds local issue drafts before they are copied into GitHub Issues. Scout lives inside the Mechanical Cupcakes OS Shell monorepo, so future GitHub issues should be created in `ErnestOfGaia/Mechanical-Cupcakes-OS-Shell` and scoped to `apps/scout`.

## How To Use These

1. Pick one issue.
2. Read the linked context docs.
3. Confirm the acceptance criteria.
4. Do only the scoped work.
5. Update the issue status when complete.
6. Later, copy mature issues into GitHub Issues for Jules or other contributors.

## GitHub Repo Context

- Repository: `https://github.com/ErnestOfGaia/Mechanical-Cupcakes-OS-Shell`
- Scout app path: `apps/scout/scout-app`
- Recommended labels: `app: scout`, `area: garage`, `good-for-jules`, `local-prototype`

## Issue Size Rule

Each issue should teach one concept and produce one concrete result. Avoid broad tasks like "build the app" or "implement protocol."

Good issue:

- Build static Garage layout from mock data.

Too broad:

- Build the whole Scout prototype.

## Status Values

- `draft` - rough idea, not ready to assign.
- `ready` - clear enough to work on.
- `in-progress` - actively being worked.
- `done` - completed and verified.
- `blocked` - needs a decision or dependency.
