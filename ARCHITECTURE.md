# Mechanical Cupcakes OS ‚Äî Architecture Reference

> Single source of truth for all technical decisions. Read this before starting any issue. Decisions documented here are made ‚Äî not open questions.

---

## System Overview

Mechanical Cupcakes OS (MCOS) is a Next.js 16 monorepo shell that unifies multiple mini-apps under a single intelligent roof. A shared AI agent (Hoot) provides OS-level navigation and knowledge search across the shell. Mini-apps run as independent Next.js apps on separate ports, embedded via iframes or linked from the shell.

---

## Monorepo Structure

```
/ (root)                         ‚Äî MCOS Shell (Next.js 16, port 3000)
  src/
    app/                         ‚Äî Shell routes (/pelican, /postcards, /ochi, /scout)
    components/shell/            ‚Äî TopBar, HootPanel, DirectoryDropdown
    mastra/                      ‚Äî Hoot agent + RAG tools
    lib/                         ‚Äî brain.ts (RAG search), embedding.ts, utils.ts
  apps/
    postcards/                   ‚Äî Visual messaging app (port 3001)
    ochi/ochi-app/               ‚Äî Hospitality dashboard (port 3003)
    scout/scout-app/             ‚Äî P2P agent discovery prototype (port 3004)
  public/
    brain.json                   ‚Äî Generated RAG embeddings (NOT committed to git)
    interstellar_garage.png      ‚Äî Scout splash page hero image
```

## Key Technology Decisions

### Shell (root `src/`)

- **Next.js 16.2.4** with App Router. All shell routes are under `src/app/`.
- **React 19.2.4** ‚Äî functional components and hooks only.
- **Tailwind CSS 4** for all shell styling. Use existing utility classes; do not change the design token setup in `globals.css`.
- **Mastra + Claude 3.5 Haiku** for the Hoot agent. Do not change the model without Ernest's approval.
- **Voyage AI + brain.json** for RAG. `brain.json` is generated by `npm run ingest` and is NOT committed to git. Run ingest after updating docs.
- **`cn()` from `@/lib/utils`** for conditional class names. Use it wherever class names are conditional.
- **No test framework** in root shell (v0.1 constraint). Tests live in sub-apps only.
- **No Redux, Zustand, or external state libraries** ‚Äî React Context is acceptable where needed.

### Scout App (`apps/scout/scout-app/`)

This app has strict constraints unique to its v0.1 prototype scope:

- **Next.js 15.2.0** ‚Äî NOT the same version as the shell. Read `node_modules/next/dist/docs/` before writing any route code.
- **Plain CSS only** ‚Äî NO Tailwind utility classes in component markup. All styles are CSS custom properties in `src/app/globals.css`. New classes go in `globals.css`, not inline styles.
- **No API routes** ‚Äî `src/app/api/` must never be created.
- **No network calls** ‚Äî no `fetch()`, no WebSocket, no real P2P traffic.
- **No browser storage** ‚Äî no `localStorage`, no `sessionStorage`, no cookies.
- **No `ethers.js`** in UI code.
- **No new npm packages** without checking with Ernest first.
- **framer-motion is installed but MUST NOT be used in v0.1.**
- **State location** ‚Äî all React state lives in `app/page.tsx` via `useState`. No context, no state libraries.
- **`[MODE: SIMULATED]` badge must always be visible** ‚Äî apply `.mode-badge` class.
- **Message envelope** ‚Äî all inter-node messages must use `ScoutEnvelope<TPayload>` from `src/lib/envelope.ts`.

### Postcards App (`apps/postcards/`)

- Next.js with NextAuth for authentication.
- Session cookies must use `sameSite: "lax"` (not `"none"`).
- S3-backed file storage. Prisma + PostgreSQL for data.

### Pellito Hub (external ‚Äî formerly `apps/pelican/`)

- Interactive recipe library / role-based kitchen AI (prep, cook, server).
- Deployed independently (Next.js 15 + SQLite) at `https://pelican.mechanicalcupcakes.fun`. Not built into the shell.
- The shell `/pelican` route embeds it via iframe (`PELICAN_URL` env var, falling back to the live subdomain). Follows the OCHI pattern below.

### Ochi App (`apps/ochi/ochi-app/`)

- Hospitality intelligence dashboard.
- The shell `/ochi` route embeds it via iframe (`OCHI_URL` env var, or port 3003).

### Embedded mini-app convention

Mini-apps render inside the shell at their route via an `<iframe>` to the standalone deployment ‚Äî see `src/app/ochi/page.tsx` (the reference) and `src/app/pelican/page.tsx`. Each embed reads a per-app `<APP>_URL` env var (e.g. `OCHI_URL`, `PELICAN_URL`) evaluated at runtime on the server, falling back to the live `*.mechanicalcupcakes.fun` subdomain when unset. These vars live in the shell's runtime `.env` (`env_file: .env`, gitignored) ‚Äî not in the compose files. New mini-apps should follow this same pattern.

## Design Systems

### MCOS Shell ‚Äî "Midnight Workshop"

- Dark background (`#0a0a0a` base) with violet (`--accent-violet`) and teal (`--teal`) accents.
- CSS custom properties defined in `src/app/globals.css`.
- Key utility classes: `.glass-panel`, `.glass-shadow`.
- Card corners: `rounded-[40px]`; panels: `rounded-2xl`.
- All shell components use Tailwind CSS 4 utility classes.
- Use `cn()` from `@/lib/utils` for conditional classes.

### Scout App ‚Äî "Garage / Mission Control"

- Deep dark navy (`#020617`) with blue accents (`#89ceff`) and amber status indicators (`#ffb95f`).
- Fonts: Work Sans (body), Space Grotesk (monospace labels/headings).
- All styles come from `apps/scout/scout-app/src/app/globals.css`.
- Key layout classes: `.garage-shell`, `.garage-workspace`, `.primary-column`, `.top-grid`.
- Key component classes: `.module`, `.module-heading`, `.console-card`, `.identity-card`, `.identity-grid`.
- Key activity classes: `.activity-feed`, `.activity-event`, `.event-type.amber/.blue/.muted`, `.event-summary`.
- Key state classes: `.whiteboard-empty`, `.empty-icon`, `.mode-badge`.

## Message Protocol (Scout)

All Scout inter-node messages follow the `ScoutEnvelope<TPayload>` shape defined in:
```
apps/scout/scout-app/src/lib/envelope.ts
```

In v0.1 simulated mode, `signature` is always `null`. Use `createScoutEnvelope()` from that file to create envelopes ‚Äî never construct the shape manually.

## Routing (Shell)

| Route | Behavior |
|-------|----------|
| `/` | Landing grid ‚Äî Pellito Hub, News Hub World, Ochi, Scout cards |
| `/pelican` | iframe to external Pellito Hub URL (`PELICAN_URL`) |
| `/newshub` | iframe to external News Hub World URL (`NEWSHUB_URL`) |

| `/scout` | Splash page with `interstellar_garage.png` hero + link to localhost:3004 |

## What NOT To Do

- Do not import `ethers.js` in Scout UI code.
- Do not add API routes to the Scout app (`src/app/api/` must not exist).
- Do not use `framer-motion` in Scout v0.1.
- Do not use Redux, Zustand, or any state management library anywhere in the shell.
- Do not use `localStorage`/`sessionStorage` in the Scout app.
- Do not commit `.env` files, `public/brain.json`, `node_modules`, `.next/`, or `*.tsbuildinfo`.
- Do not install new npm packages in the Scout app without Ernest's sign-off.
- Do not use Tailwind utility classes in Scout component markup.
- Do not hardcode the `[MODE: SIMULATED]` badge away ‚Äî it must always render.
- Do not add features beyond the acceptance criteria of the issue. Note extras in the PR description instead.
